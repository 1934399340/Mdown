# Mdown v3.0 PRD（一页版）

> 产品定位：安卓端 WYSIWYG Markdown 编辑器，内置 AI 写作助手  
> 版本日期：2026-06-10 | 状态：Beta

---

## 一、目标用户 & 痛点

| 用户 | 一句话画像 | 核心痛点 |
|------|----------|---------|
| 📝 学生/学者 | 上课记笔记，想用 Markdown 但不会语法 | 纯文本编辑器门槛高，格式操作靠记忆 |
| 💻 技术作者 | 写 README、技术博客，需要随时编辑 | 安卓端没一个好用的编辑器，导出格式少 |
| 📊 职场人 | 写周报、会议纪要，追求效率 | 写作时频繁切换 App 查资料/润色，打断心流 |
| 📱 移动办公族 | 通勤时处理文档，到达后导出分享 | 手机编辑体验差，想导出 PDF 要传电脑 |

**核心洞察**：安卓端没有一款编辑器能同时做到 **"上手即写 + AI 辅助 + 自由导出"**。

---

## 二、核心流程

```
新建/打开文件 → 选择编辑模式 → 写作 + AI 辅助 → 导出/保存
                    │                    │
                    ├─ 编辑（WYSIWYG）    ├─ 输入"写一篇周报"→ AI 生成
                    ├─ 源代码             ├─ 选中段落"润色"→ AI 改写
                    └─ 分屏预览           └─ 输入"删掉第二段"→ AI 全文修改
```

---

## 三、功能优先级

**P0（必须有，v1.0 已交付）**
- Markdown 编辑与实时渲染
- 文件创建/保存/删除/搜索
- WYSIWYG 编辑模式（contentEditable WebView）

**P1（v3.0 已交付）**
- AI 写作助手（三种模式：新建 / 选中修改 / 全文自动修改）
- 5 格式导出（PDF / HTML / PNG / TXT / MD）
- 深色模式
- 格式化工具栏（30+ 按钮，25+ 快捷键）

**P2（下一版本）**
- AI 流式输出（SSE，边生成边显示）
- 多模型支持（OpenAI / Anthropic）
- 文档模板（周报、会议纪要等）

**P3（未来）**
- 云端同步（Google Drive）
- 数学公式渲染（LaTeX）

## 四、指标设计

> 注：当前 Beta 阶段，以下为上线后的核心指标框架。

| 类别 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| 📊 激活 | 次日留存 | > 40% | 验证"编辑器 + AI"是否有持续价值 |
| 📊 激活 | 首日 AI 使用率 | > 50% | AI 是否被用户发现并使用 |
| 📊 激活 | 首次导出时间 | < 首次使用时长的 50% | 用户是否完成了"写完→导出"闭环 |
| ✍️ 功能 | AI 模式分布 | 新建 40% / 修改 60% | 验证"修改已有内容"是否比"生成新内容"更有价值 |
| ✍️ 功能 | AI 生成后保存率 | > 80% | AI 输出的内容质量是否合格 |
| 😊 体验 | 编辑器崩溃率 | < 0.5% | WebView 大文档场景稳定性 |
| 😊 体验 | 导出成功率 | > 98% | 5 种导出引擎可靠性 |
